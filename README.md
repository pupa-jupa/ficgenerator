# FicGenerator

FicGenerator — браузерное приложение для генерации фанфиков и оригинальных историй с помощью Google Gemini. Проект работает как статический сайт: без сборки, сервера и тяжелых зависимостей.

## Возможности

- Генерация историй по фандому, пейрингам, жанрам, рейтингу, персонажам и авторским заметкам.
- Поддержка оригинальных историй и фанфиков с учетом канона.
- Загрузка изображения-референса: картинка передается в Gemini как визуальный контекст.
- Генератор сюжетных планов по краткой идее.
- Встроенный список случайных идей, фандомов, пейрингов и сюжетных зацепок.
- Продолжение, переписывание и точечное редактирование уже созданного текста.
- Личная библиотека историй в браузере через `localforage`.
- Экспорт истории в PDF через системную печать браузера.
- Генерация обложек через Pollinations.ai без отдельного API-ключа.
- Статистика текста: слова, символы, абзацы, примерное время чтения и другие показатели.

## Технологии

- HTML, CSS и Vanilla JavaScript.
- ES-модули в папке `js/`.
- Google Gemini API для генерации текста.
- `marked` для Markdown-рендера.
- `DOMPurify` для очистки HTML.
- `localforage` для локального хранения библиотеки.
- Pollinations.ai для генерации обложек.

CDN-зависимости подключаются прямо в `index.html`.

## Быстрый старт

1. Скачайте или клонируйте репозиторий.
2. Откройте `index.html` в современном браузере.
3. Перейдите во вкладку «Настройки».
4. Вставьте API-ключ Google Gemini.
5. Выберите модель и сохраните настройки.
6. Вернитесь во вкладку создания истории и запустите генерацию.

API-ключ хранится только в `sessionStorage`, то есть действует в рамках текущей сессии браузера. Выбранная модель сохраняется в `localStorage`.

## API-ключ Gemini

Ключ можно создать в [Google AI Studio](https://aistudio.google.com/app/apikey).

Приложение обращается к API напрямую из браузера:

```text
https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent
```

Поэтому не публикуйте личный ключ в коде, README, скриншотах или публичных настройках.

## Модели

По умолчанию используется `gemini-2.5-flash`. Также интерфейс поддерживает выбор более быстрой lite-модели, если она доступна в настройках приложения.

```text

    ├── covers.js       # Генерация и скачивание обложек

    ├── library.js      # Локальная библиотека и PDF-экспорт
    ├── main.js         # Точка входа и обработчики интерфейса
```

Актуальная точка входа — `js/main.js`, подключенная в `index.html` как `type="module"`.

## Локальное хранение

Приложение использует браузерное хранилище:

- `sessionStorage.gemini_api_key` — API-ключ на время текущей сессии.
- `localStorage.gemini_model` — выбранная модель.
- `localforage.fanfic_library` — сохраненные истории.

Очистка данных сайта в браузере удалит сохраненные истории и настройки.

## Работа с историями

Основной сценарий:

1. Выберите тип истории: фанфик или оригинальная работа.
2. Заполните фандом, пейринги, жанры, персонажей и описание сюжета.
3. При необходимости добавьте референс-изображение.
4. Нажмите «Сгенерировать».
5. После генерации сохраните текст в библиотеку, продолжите его или отредактируйте.
6. При необходимости скачайте PDF или создайте обложку.

## Разработка

Проект не требует установки npm-зависимостей для запуска. `package.json` содержит только метаданные и ограничение Node.js для совместимости хостингов.

Если нужен локальный сервер вместо открытия файла напрямую, можно запустить любой статический сервер, например:

```bash
npx serve .
```

После этого откройте адрес, который покажет сервер.

## Деплой

Проект можно размещать на любом статическом хостинге:

- Vercel
- Netlify
- GitHub Pages
- Cloudflare Pages
- любой сервер, отдающий статические файлы

Для деплоя достаточно опубликовать `index.html`, `style.css` и папку `js/`.

## Безопасность

- Не коммитьте API-ключи.
- Не храните чужие личные данные в библиотеке историй.
- Помните, что запросы к Gemini выполняются из браузера пользователя.
- Если проект станет публичным сервисом, лучше вынести обращения к Gemini на backend-прокси.
